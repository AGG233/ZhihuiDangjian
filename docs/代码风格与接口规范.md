# 代码风格与接口规范

## API 路径规范

所有接口路径统一以 `/api` 开头：

| 模块 | 前缀 | 示例 |
|------|------|------|
| 用户端 | `/api/<module>/...` | `/api/user/users`, `/api/course/...` |
| 管理端 | `/api/admin/<module>/...` | `/api/admin/users`, `/api/admin/course/...` |
| 公共接口 | `/api/...` | `/api/school/all` |

路径片段全部使用复数形式：`/users/{userId}` 而非 `/user/{userId}`，`/chapters/{chapterId}` 而非 `/chapter/{chapterId}`。

认证模块路径为 `/api/auth/...`（非旧 `/auth`）。

## 分层命名规范

### Controller

- 用户端 Controller：`UserXxxController`，放 `controller/user/` 包下
- 管理端 Controller：`AdminXxxController`，放 `controller/admin/` 包下
- 纯公共接口：不放 user/admin 子包，用业务名
- 所有接口返回 `Result<T>`，SSE 流接口除外（返回 `Flux<T>`）
- 所有请求体使用 `@RequestBody @Valid XxxRequest`
- 所有接口保留中文 `@Tag`、`@Operation`、`@Schema`

### 请求/响应模型

- **请求体**：`pojo/request/XxxRequest`，使用 `@Data @Builder @Schema`
- **响应体**：`pojo/response/XxxResponse`，使用 `@Data @Builder @Schema`
- **Entity**：仅表示数据库表，不作为公开接口响应模型直接返回
- **禁止新增** `Dto`、`VO` 后缀作为公开 API 类型（旧类型已迁移完成）
- **内部查询条件**：非公开 API 的内部模型可保留 `pojo/dto/`

### Service

- 业务失败统一抛 `BusinessException(errorCode, message)`
- 禁止用 `return false`、`return null` 表示业务失败
- 查询继续使用 MyBatis-Plus `LambdaQueryWrapper`
- 方法签名明确：`void create()`, `void update()`, `void delete()`, `XxxResponse get()`

### ErrorConstants

各模块错误码使用模块级常量类，互不重叠：

| 模块 | 常量类 | 范围 |
|------|--------|------|
| 通用 | `ErrorConstants` | 1-99 |
| auth | `AuthErrorConstants` | 1000-1999 |
| user | `UserErrorConstants` | 2000-2999 |
| category | `CategoryErrorConstants` | 3000-3099 |
| chapter | `ChapterErrorConstants` | 3100-3199 |
| course | `CourseErrorConstants` | 3200-3299 |
| learning | `LearningErrorConstants` | 4000-4999 |
| resource | `ResourceErrorConstants` | 5000-5999 |
| quiz | `QuizErrorConstants` | 6000-6999 |
| graph | `GraphErrorConstants` | 7000-7999 |
| ai | `AiErrorConstants` | 8000-8999 |

### 转换器

- 使用 MapStruct，接口名 `XxxConvertor`（保留项目惯例 `Convertor` 拼写）
- 包名统一 `pojo/convertor`
- 方法命名：`toEntity()`、`toRequest()`、`toResponse()`
- 转换器方法返回 `List` 时命名为 `toResponseList()`、`toRequestList()`

### 包名规范

- 基础包：`com.rauio.smartdangjian.server.<module>.<layer>`
- 常量包：统一 `constants`（非 `Constant`）
- 转换器包：统一 `convertor`（非 `conventor`）
- 所有包名小写，禁止大写包名
- search 模块包：`com.rauio.smartdangjian.server.search`（含 `server` 层级）

## 权限规范

使用现有 AOP 注解体系：

- `@PermissionAccess(UserType.XXX)` — 接口级别角色控制
- `@RequireUser` — 要求已登录用户
- `@ResourceAccess` — 资源级权限
- `@DataScopeAccess` — 数据范围过滤

禁止新增另一套权限机制。

## 格式化规范

- Spotless + palantir-java-format（4 空格缩进）
- `./gradlew spotlessApply` 自动格式化
- `check` 任务依赖 `spotlessCheck`，格式化不通过则构建失败
- import 顺序：`java` → `javax` → `jakarta` → `org` → `com`

## 禁止事项

- 禁止新增无泛型 `Result`（必须 `Result<T>`）
- 禁止新增直接拼接 SQL 字符串；现有 `.last("limit 1")` 可保留，新增查询优先用分页或 Wrapper
- 禁止新增裸 JSON 字符串测试请求，优先用测试数据工厂或 ObjectMapper
- 禁止 Controller 直接返回 Entity 类型作为响应
- 禁止 Service 层用 boolean 返回值表示业务失败
