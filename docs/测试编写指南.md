# 测试编写指南

## 1. 文档目的

本文档说明项目的测试基础设施、编写规范与常用模式，供开发人员编写新测试时参考。

当前测试覆盖范围：
- 管理员用户分页搜索接口 `POST /api/admin/users/search`（正常、异常、边界、安全）
- 管理员用户详情接口 `GET /api/admin/users/{id}`
- 权限切面 (`UserAspect`)
- 数据范围切面 (`UserManagementAspect`)

## 3. 测试类型与模式

项目使用两种测试类型：

### 3.1 MockMvc 集成测试（Controller 层）

使用 Spring Boot 测试上下文，Mock 掉 Service 层，通过 MockMvc 发送 HTTP 请求验证 Controller 行为。

```java
@SpringBootTest(webEnvironment = WebEnvironment.MOCK, classes = XxxTest.TestConfig.class)
@AutoConfigureMockMvc(addFilters = false)  // 禁用 Spring Security 过滤器链
class XxxTest {

    @Autowired
    private MockMvc mockMvc;

    @MockitoBean
    private XxxService xxxService;  // Mock 掉 Service 层

    @BeforeEach
    void setUpSecurityContext() {
        // 手动设置 SecurityContext，模拟已登录用户
        CurrentUserPrincipal principal = ...;
        SecurityContextHolder.getContext().setAuthentication(
            new UsernamePasswordAuthenticationToken(principal, null, emptyList())
        );
    }
}
```

**TestConfig 内联配置类**：通过 `@SpringBootConfiguration` + `@EnableAutoConfiguration(exclude = {...})` + `@ComponentScan` 构建最小化 Spring 上下文，只扫描 controller 包，排除数据源、JPA、安全自动配置和事务管理器。

**关键行为**：
- `addFilters = false` 禁用了 Spring Security 过滤器链，但仍可通过 `SecurityContextHolder` 设置当前用户
- `WebConfig`（注册 `UserAspect`）因未被排除而**仍然生效**，`@PermissionAccess` 权限检查在工作
- `DataScopeAccessAspect` 未被扫描（不在 `@ComponentScan` 范围内），数据范围限制不生效

### 3.2 Service 层单元测试（AI 模块模式）

AI 模块的 Service 继承自 MyBatis-Plus `ServiceImpl`，测试时需用 `@Spy` + `doReturn()` mock 父类方法：

```java
@ExtendWith(MockitoExtension.class)
class PromptServiceTest {

    @Mock
    private AiPromptsMapper mapper;

    @Spy
    @InjectMocks
    private PromptService promptService;  // extends ServiceImpl

    @Test
    @DisplayName("空提示词列表返回默认系统提示词")
    void emptyPromptsReturnsDefaultSystemPrompt() {
        // mock 父类 ServiceImpl 的 listEnabledSystemPrompts 方法
        doReturn(Collections.emptyList()).when(promptService).listEnabledSystemPrompts("CHAT");

        String prompt = promptService.buildSystemPrompt("CHAT");

        assertThat(prompt).contains("智慧党建平台AI助手");
    }
}
```

**关键规则**：
- 用 `@Spy` + `@InjectMocks`，不能用纯 `@InjectMocks`（否则 ServiceImpl 的 `list()`/`getById()`/`save()` 会因 MyBatis-Plus 未初始化而抛异常）
- `@DisplayName` 必须用中文描述测试意图
- 断言使用 AssertJ：`assertThat(actual).isEqualTo(expected)`

### 3.3 POJO/Entity 单元测试

不依赖任何框架，直接测试实体方法：

```java
class AiSkillTest {

    @Test
    @DisplayName("YAML 特殊字符被转义")
    void yamlSpecialCharsEscaped() {
        AiSkill skill = AiSkill.builder()
                .name("skill: test")
                .description("描述包含 \"引号\"")
                .content("content")
                .build();

        String result = skill.renderSkillMarkdown();

        assertThat(result).contains("name: \"skill: test\"");
        assertThat(result).contains("\\\"引号\\\"");
    }
}
```

### 3.4 纯单元测试（AOP 切面）

不依赖 Spring 上下文，使用 `@ExtendWith(MockitoExtension.class)` 直接测试 AOP 切面的方法逻辑。

```java
@ExtendWith(MockitoExtension.class)
class XxxSecurityTest {

    // 直接实例化切面类
    private final UserAspect userAspect = new UserAspect();

    @Mock
    private UserMapper userMapper;

    @BeforeEach
    void setUp() {
        // 手动组装依赖
        userManagementAspect = new UserManagementAspect(userMapper);
    }

    @Test
    void testSomething() {
        // 手动设置 SecurityContext
        setSecurityContext("user-1", UserType.STUDENT, "uni-1");

        // 构造 mock JoinPoint
        JoinPoint jp = mockJoinPointForMethod(...);

        // 直接调用切面方法
        assertThatThrownBy(() -> userAspect.checkPermissionAccess(jp))
                .isInstanceOf(BusinessException.class);
    }
}
```

**适用场景**：测试 AOP 切面的权限逻辑、数据范围注入逻辑，这些在 MockMvc 集成测试中因 component-scan 限制无法覆盖。

## 4. 测试数据工厂

`UserTestDataFactory` 是纯静态工厂类，提供可复用的测试数据构建方法，避免测试间数据构造重复。

### 4.1 单用户构建

```java
UserVO vo = UserTestDataFactory.createDefaultUserVO();
// id="user-001", username="zhangsan", realName="张三", userType=STUDENT
// email="zhangsan@example.com", partyStatus=FORMAL_MEMBER, status=ACTIVE ...

UserVO school = UserTestDataFactory.createSchoolUserVO();
UserVO manager = UserTestDataFactory.createManagerUserVO();
UserVO inactive = UserTestDataFactory.createInactiveUserVO();
UserVO banned = UserTestDataFactory.createBannedUserVO();

UserVO custom = UserTestDataFactory.createUserVO("id", "name", "姓名", UserType.STUDENT);
```

### 4.2 批量构建

```java
// 10 个 STUDENT 用户，id 为 user-001 到 user-010，字段有变化
List<UserVO> list = UserTestDataFactory.createUserVOList(10);

// 指定类型和学校
List<UserVO> schoolUsers = UserTestDataFactory.createUserVOList(5, UserType.SCHOOL, "uni-001");
```

### 4.3 分页构建

```java
Page<UserVO> page = UserTestDataFactory.createPage(records, current, size, total);
Page<UserVO> empty = UserTestDataFactory.createEmptyPage(current, size);
```

### 4.4 JSON 工具

`UserTestDataFactory.toJson(UserDto)` 使用 Jackson `ObjectMapper` 序列化 DTO，正确输出 `@JsonValue` 枚举的中文表示（如 `UserType.MANAGER` → `"管理员"`），不要手动拼接 JSON 字符串。

```java
UserDto dto = UserTestDataFactory.createSearchDto();
dto.setUsername("test");
dto.setUserType(UserType.STUDENT);
String json = UserTestDataFactory.toJson(dto);
// → {"username":"test","userType":"学生"}
```

## 5. 关键注意事项

### 5.1 枚举序列化

以下枚举使用 `@JsonValue` 序列化，JSON 中必须使用中文/原始值，不可使用枚举名：

| 枚举 | 枚举名 | JSON 值 |
|------|--------|---------|
| `UserType` | `STUDENT` | `"学生"` |
| `UserType` | `SCHOOL` | `"学校"` |
| `UserType` | `MANAGER` | `"管理员"` |
| `PartyStatus` | `FORMAL_MEMBER` | `"正式党员"` |
| `PartyStatus` | `PROBATIONARY_MEMBER` | `"预备党员"` |
| `PartyStatus` | `DEVELOPMENT_TARGET` | `"发展对象"` |
| `PartyStatus` | `PARTY_ACTIVIST` | `"积极分子"` |
| `PartyStatus` | `GENERAL_PUBLIC` | `"群众"` |
| `AccountStatus` | `ACTIVE` | `"active"` |
| `AccountStatus` | `INACTIVE` | `"inactive"` |
| `AccountStatus` | `BANNED` | `"banned"` |

### 5.2 数据脱敏

`UserVO` 的 `email` 和 `phone` 字段标注了 `@Sensitive`，Jackson 序列化时会自动脱敏：

- `zhangsan@example.com` → `zh*n@example.com`
- 手机号仅对 11 位纯数字做脱敏，带 `+86` 前缀的不处理

测试断言中需使用脱敏后的值。

### 5.3 email 字段的序列化陷阱

`UserDto.email` 也标注了 `@Sensitive`（用于序列化），其 `SensitiveDataSerializer.desensitizeEmail()` 在值不含 `@` 符号时会抛出 `StringIndexOutOfBoundsException`。

因此构造 `UserDto` 请求体时，若设置 email 字段，值必须包含 `@` 且有非空前缀（如 `"user@example.com"`），否则 `toJson()` 会失败。

### 5.4 `@Nested` 与 `@BootstrapWith` 冲突

`@Nested` 内部类会继承外部类的注解。若外部类同时有 `@BootstrapWith`（显式）和 `@SpringBootTest`（内含隐式 `@BootstrapWith`），Spring 会检测到两个冲突的 `@BootstrapWith` 声明导致 `IllegalStateException`。

**解决方案**：不要在外层测试类使用显式 `@BootstrapWith`。`@SpringBootTest(classes = TestConfig.class)` 中的 `classes` 属性已指定配置源，无需自定义 Bootstrapper。

## 6. 异常响应对照

| 异常类型 | HTTP 状态码 | `$.code` | 说明 |
|----------|------------|----------|------|
| `BusinessException(1005, "msg")` | 400 | `"1005"` | 业务异常 |
| `BusinessException(4003, "msg")` | 400 | `"4003"` | 权限不足 |
| `IllegalArgumentException` | 400 | `"400"` | 参数校验失败（含无效枚举值） |
| `IllegalStateException` | 409 | `"409"` | 状态冲突（如技能缓存加载失败） |
| `RuntimeException` | 500 | `"500"` | 通用运行时异常 |
| 请求体缺失/格式错误 | 400 | `"400"` | JSON 解析失败 |
| 参数类型转换失败 | 500 | — | 如 `pageNum=abc` |

## 8. 新增测试检查清单

编写新 Controller 测试时：

- [ ] 创建/复用 `TestConfig` 内联配置类，排除不必要的自动配置
- [ ] 使用 `@MockitoBean` Mock 掉 Service 层
- [ ] `@BeforeEach` 中设置合适的 `SecurityContext`（当前用户类型）
- [ ] 使用 `UserTestDataFactory` 构造 Mock 返回数据
- [ ] 使用 `UserTestDataFactory.toJson()` 构造请求体（处理 `@JsonValue`）
- [ ] 覆盖：正常返回、空结果、业务异常、参数错误、分页边界、XSS/SQL 注入
- [ ] 涉及 email/phone 断言时注意脱敏值

编写 AOP 安全测试时：

- [ ] 使用 `@ExtendWith(MockitoExtension.class)`，不启动 Spring 上下文
- [ ] 在接口/方法上声明带注解的方法供反射获取
- [ ] 使用 `lenient().when()` 避免 `UnnecessaryStubbingException`
- [ ] 测试方法直接调用切面的 public 方法，手动设置 `SecurityContext`
