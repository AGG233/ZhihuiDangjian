## 4. 模块划分与关键代码

## 4.1 个性化学习推荐模块

### 4.1.1 模块职责

该模块根据用户的学习行为数据、章节学习记录、用户相似度等信息，为用户推荐可能感兴趣的课程内容。当前实现以 **协同过滤推荐** 为主，并保留了全文检索与推荐融合的混合搜索思路。

### 4.1.2 关键代码位置

- `services/search/src/main/java/com/rauio/smartdangjian/search/service/RecommendService.java`
- `services/search/src/main/java/com/rauio/smartdangjian/search/service/SearchService.java`
- `services/user/src/main/java/com/rauio/smartdangjian/user/service/UserSimilarityService.java`

### 4.1.3 实现说明

`RecommendService` 的核心思想是：

1. 基于用户学习行为构建“用户-章节”关系
2. 计算用户之间的相似度
3. 找到与目标用户最相近的用户集合
4. 汇总相似用户已学习而目标用户尚未学习的课程
5. 按得分排序后生成推荐结果

### 4.1.4 关键代码示例

```java
public Page<String> recommendByCF(String userId,int pageNum, int pageSize) {
    int neighborSize = 10;
    Page<UserSimilarity> similarityPage = userSimilarityMapper.selectPage(
            new Page<>(1, neighborSize),
            new LambdaQueryWrapper<UserSimilarity>()
                    .eq(UserSimilarity::getUserId1, userId)
                    .orderByDesc(UserSimilarity::getSimilarityScore)
    );
    ...
}
```

上面代码体现了推荐模块首先从 `user_similarity` 数据中取出与当前用户最相近的用户集合，再进一步生成推荐课程列表。

### 4.1.5 论文可用表述

> 个性化学习推荐模块以用户学习记录和用户相似度为基础，采用协同过滤算法对学习资源进行排序推荐，从而增强课程分发的针对性与个体适配能力。

## 4.2 AI 智能问答模块

### 4.2.1 模块职责

该模块是系统智能化能力的核心，基于 **Spring AI + Alibaba Cloud AI Graph（ReactAgent 框架）** 构建，提供：

- AI 文本问答（党务学习场景下的日常问答）
- AI 学习评估（结合学习记录和答题情况生成评估报告）
- AI 主题出题（围绕指定主题生成测试小题）
- 多 Agent 协作（CHAT / QUIZ / EVALUATION 三种 Agent，各有独立系统提示词和工具集）
- 工具增强调用（学习记录、答题记录、用户画像、课程推荐等 6 个 Tool）
- Prompt 与技能数据库管理
- 跨会话长期记忆（MySQL 持久化 + Redis 检查点状态）

### 4.2.3 架构设计

模块采用 **Agent 注册表模式**，三种 Agent 统一由 `AiAgentRegistry` 管理：

```java
public enum AiAgentType {
    CHAT("CHAT", "chat-agent", "处理党务学习场景下的日常问答"),
    QUIZ("QUIZ", "quiz-agent", "根据主题和学习记录生成测试小题"),
    EVALUATION("EVALUATION", "evaluation-agent", "结合学习记录和答题情况给出学习评估");
}
```

每个 Agent 共享同一套 ChatModel 和工具集，但通过 `DynamicSystemPromptInterceptor` 动态注入不同的系统提示词和长期记忆。

### 4.2.4 控制层入口

`UserChatController` 提供三个 SSE 流式接口，所有响应以 `Flux<AiChatResponse>` 流式返回：

```java
@PermissionAccess(UserType.STUDENT)
@RestController
@RequestMapping("/api/ai/chat")
public class UserChatController {

    @PostMapping(produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<AiChatResponse> chat(@RequestBody @Valid AiChatRequest request) {
        return llmService.chat(request);
    }

    @PostMapping(value = "/evaluation", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<AiChatResponse> evaluate(@RequestBody @Valid AiEvaluationRequest request) {
        return llmService.evaluate(request);
    }

    @PostMapping(value = "/quiz", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<AiChatResponse> quiz(@RequestBody @Valid AiQuizRequest request) {
        return llmService.quiz(request);
    }
}
```

流式管道中内建了错误恢复（`onErrorResume`）、超时控制（`timeout(120s)`）、断连处理（`doOnCancel`）和速率限制。

### 4.2.5 工具增强调用

Agent 可通过 `@Tool` 注解的方法调用 6 个业务工具获取真实数据：

| 工具 | 功能 |
|------|------|
| `UserInfoTool` | 获取当前用户基本信息 |
| `UserProfileTool` | 获取用户学习画像（统计、知识掌握度、兴趣分类） |
| `LearningTool` | 查询用户学习记录（按时间/课程/章节） |
| `UserQuizAnswerTool` | 查询用户答题记录 |
| `QuizTool` | 获取当前会话的出题思路 |
| `RecommendTool` | 基于协同过滤+知识图谱+用户画像的个性化课程推荐 |

所有工具通过 `ToolContext` 传递用户身份，支持 `parallelToolExecution(true)` 并行调用，最大并行度为 4。

### 4.2.6 动态提示词与技能系统

提示词（`AiPrompts`）和技能（`AiSkill`）存储在 MySQL 中，通过管理后台 CRUD：

- `AdminPromptController` — 系统提示词的 CRUD（需 MANAGER 权限）
- `AdminSkillController` — AI 技能的 CRUD（需 MANAGER 权限）

`DynamicSystemPromptInterceptor` 在每次 Agent 调用时动态组装系统提示词，并从数据库中加载技能列表注入 Agent 上下文。`DatabaseSkillRegistry` 使用 volatile + 原子替换保证技能缓存的线程安全。

### 4.2.7 会话与记忆

采用双层记忆架构：

- **Redis**（`RedisSaver`）— 保存 Agent 对话的检查点状态，支持会话恢复
- **MySQL**（`AiChatMessage`）— 每次对话完成后保存一问一答，用于跨会话长期记忆

`AiMemoryService.buildLongTermMemory()` 查询用户历史对话（排除当前会话），按时间整理为结构化记忆注入到系统提示词中。

### 4.2.8 论文可用表述

> AI 智能问答模块基于 Spring AI 与 ReactAgent 框架构建，采用多 Agent 注册表模式统一管理对话、出题和评估三类智能代理。模块通过工具增强调用机制整合学习记录、答题记录和用户画像等业务数据，结合数据库驱动的动态提示词与技能系统，以及双层记忆架构（Redis 会话状态 + MySQL 长期记忆），实现对党务学习场景的上下文感知智能交互。

public Boolean create(UserLearningRecordDto dto) {

- `src/main/resources/application.yaml`

### 4.4.3 图谱写入实现

核心写入逻辑如下：

```java
String cypher = """
        MERGE (u:User {id:$userId})
        SET u.name = $userName
        MERGE (c:Course {id:$courseId})
        SET c.title = $courseTitle
        MERGE (ch:Chapter {id:$chapterId})
        SET ch.title = $chapterTitle
        MERGE (u)-[:LEARNED]->(c)
        MERGE (c)-[:HAS_CHAPTER]->(ch)
        MERGE (u)-[:LEARNED_CHAPTER]->(ch)
        """;
```

该实现说明系统使用的是 **图数据库知识图谱建模方式**，以节点和关系形式描述学习档案。

### 4.9.3 关键实现

认证服务登录逻辑如下：

```java
public LoginResponse login(LoginRequest loginRequest) {
    if (!captchaService.validate(loginRequest.getCaptchaUUID(), loginRequest.getCaptchaCode())) {
        throw new BusinessException(ARGS_ERROR, "验证码错误");
    }

    Authentication authentication = authenticationManager.authenticate(
            new UsernamePasswordAuthenticationToken(
                    loginRequest.getPassport(),
                    loginRequest.getPassword()
            )
    );

    User user = (User) authentication.getPrincipal();
    String accessToken = jwtService.generateAccessToken(user, loginRequest.getPlatform());

    return LoginResponse.builder().accessToken(accessToken).build();
}
```

该流程说明系统采取了“验证码 + 账户密码 + JWT”的组合认证模式。

### 4.9.4 安全自动配置

`SecurityCoreAutoConfiguration` 中配置了认证提供器、JWT 过滤器与 CORS 策略，是系统权限控制的核心基础设施。

### 4.9.5 论文可用表述

> 认证与权限安全模块采用基于 Spring Security 的认证体系，实现验证码校验、账户密码认证、JWT 令牌签发与请求过滤，并结合角色权限控制机制保障接口访问安全。

## 6. 论文写作建议

### 6.1 关于“区块链档案”表述

如果论文中已经写了“区块链档案模块”，建议谨慎处理。因为当前代码实现明确采用的是 **Neo4j 图数据库**，而不是区块链存证技术。
